# API & Data Model Reference

## Cloud Functions API

All Cloud Functions are TypeScript with Firebase SDK v11+.

### Scheduled Functions

#### `sendSMSReminders`
**Type**: Pub/Sub Scheduled  
**Trigger**: Every 1 minute  
**Runtime**: ~10-30 seconds  

**What it does**:
1. Queries all reminders where `status = 'pending'` and `reminderTime <= now`
2. Sends SMS via Twilio to each reminder's phoneNumber
3. Updates reminder status to 'sent' or 'failed'
4. Creates audit log in `sentMessages` collection

**Environment Variables Used**:
- `process.env.TWILIO_ACCOUNT_SID`
- `process.env.TWILIO_AUTH_TOKEN`
- `process.env.TWILIO_PHONE_NUMBER`

**Success Response**: No return value (runs silently)

**Error Handling**:
- Failed SMS: status set to 'failed', failure reason logged
- Skipped if Twilio config missing
- Errors logged to Cloud Logging

**Example Log Entry**:
```
SMS sent successfully: SMxxxxxxxxxxxxxxxx
reminderId: doc123
volunteerId: user456
eventId: event789
```

### Callable Functions (Called from Frontend)

#### `createRemindersForEvent`
**Type**: HTTPS Callable  
**Authentication**: Required (user must be logged in)  
**Authorization**: Admin only  

**Request Parameters**:
```javascript
{
  eventId: string,           // Required: Firestore event document ID
  hoursBeforeEvent: number,  // Required: 1-168 hours
  message: string            // Required: SMS message text (max 160 chars recommended)
}
```

**Example Usage** (from frontend):
```typescript
import { httpsCallable } from 'firebase/functions';
import { functions } from '../config/firebase';

const createRemindersFunction = httpsCallable(functions, 'createRemindersForEvent');
const result = await createRemindersFunction({
  eventId: 'event123',
  hoursBeforeEvent: 24,
  message: 'Reminder: Volunteer event tomorrow at 2pm'
});

console.log(`Created ${result.data.remindersCreated} reminders`);
```

**Response**:
```typescript
{
  success: boolean,
  remindersCreated: number,
  reminders: string[]  // Array of created reminder document IDs
}
```

**Errors**:
- `unauthenticated`: User not logged in
- `permission-denied`: User is not an admin
- `invalid-argument`: Missing required fields
- `not-found`: Event doesn't exist
- `internal`: Other errors (see message for details)

**Permissions Check**:
1. Verifies user is authenticated
2. Gets user's admin document
3. Returns error if not admin

## Firestore Data Model

### Collections

#### `volunteers`
**Document ID**: User's Firebase UID  
**Access**: Admins (full), Self (read/write own), Others (none)

**Document Structure**:
```typescript
{
  // User identification
  uid: string;                    // Firestore document ID (same as Firebase UID)
  email: string;                  // From Firebase Auth
  name: string;                   // Full name
  phoneNumber: string;            // E.164 format: +1234567890

  // Profile information
  address?: string;               // Optional: street address
  availableHours?: number;        // Optional: hours available per month
  joinedDate: Timestamp;          // When volunteer joined
  
  // System fields
  createdAt: Timestamp;           // Account creation timestamp
  updatedAt?: Timestamp;          // Last profile update
}
```

**Example Document**:
```
volunteers/
  └── user_uid_123456
      ├── uid: "user_uid_123456"
      ├── email: "john@example.com"
      ├── name: "John Smith"
      ├── phoneNumber: "+14155552671"
      ├── address: "123 Main St, NYC, NY 10001"
      ├── availableHours: 40
      ├── joinedDate: Timestamp(1700000000, 0)
      └── createdAt: Timestamp(1700000000, 0)
```

#### `serviceEvents`
**Document ID**: Auto-generated by Firestore  
**Access**: Admins (full), Assigned volunteers (read-only), Others (none)

**Document Structure**:
```typescript
{
  // Event details
  topic: string;                      // Event name/title
  description?: string;               // Detailed description
  eventDateTime: Timestamp;           // When event occurs
  location?: string;                  // Where event takes place

  // Assignments
  assignedVolunteers: string[];       // Array of volunteer UIDs assigned
  status: 'scheduled' | 'ongoing' | 'completed' | 'cancelled';

  // System fields
  createdAt: Timestamp;               // When event was created
  updatedAt: Timestamp;               // Last modification
  createdBy?: string;                 // UID of admin who created
}
```

**Example Document**:
```
serviceEvents/
  └── event_doc_id_789
      ├── topic: "Community Cleanup"
      ├── description: "Spring park cleanup and landscaping"
      ├── eventDateTime: Timestamp(1710000000, 0)
      ├── location: "Central Park, NYC"
      ├── assignedVolunteers: ["user1", "user2", "user3"]
      ├── status: "scheduled"
      ├── createdAt: Timestamp(1699000000, 0)
      ├── updatedAt: Timestamp(1699500000, 0)
      └── createdBy: "admin_uid_456"
```

#### `reminders` ⚠️ ADMIN ONLY
**Document ID**: Auto-generated by Firestore  
**Access**: Admins (read/write), Cloud Functions (read/write), Volunteers (none)  
**Frontend Access**: BLOCKED by Firestore rules

**Document Structure**:
```typescript
{
  // Reference fields
  eventId: string;                    // Which event this reminder is for
  volunteerId: string;                // Which volunteer to remind
  
  // Contact information
  phoneNumber: string;                // E.164 format for SMS
  message: string;                    // SMS message body

  // Timing
  reminderTime: Timestamp;            // When to send reminder
  hoursBeforeEvent: number;           // Hours before event (informational)

  // Status tracking
  status: 'pending' | 'sent' | 'failed';
  sentAt?: Timestamp;                 // When SMS was sent
  retryCount?: number;                // How many times retry was attempted
  failureReason?: string;             // Error message if failed

  // System fields
  createdAt: Timestamp;               // When reminder was created
  twilioSid?: string;                 // Twilio message SID if sent
}
```

**Example Document**:
```
reminders/
  └── reminder_doc_id_456
      ├── eventId: "event_doc_id_789"
      ├── volunteerId: "user1"
      ├── phoneNumber: "+14155552671"
      ├── message: "Reminder: Community Cleanup tomorrow at 9am!"
      ├── reminderTime: Timestamp(1709996400, 0)  // 1 hour before event
      ├── hoursBeforeEvent: 24
      ├── status: "pending"  // or "sent" / "failed"
      ├── sentAt: Timestamp(1709996500, 0)
      ├── retryCount: 0
      ├── failureReason: null
      ├── createdAt: Timestamp(1699500000, 0)
      └── twilioSid: "SM..."
```

#### `sentMessages` 📋 AUDIT LOG
**Document ID**: Auto-generated by Firestore  
**Access**: Admins (read-only), Cloud Functions (write-only), Volunteers (none)

**Purpose**: Immutable audit trail of all SMS sends

**Document Structure**:
```typescript
{
  // Reference fields
  reminderId: string;                 // Which reminder generated this message
  volunteerId: string;                // Which volunteer received it
  eventId: string;                    // Which event it was for

  // Message details
  phoneNumber: string;                // Recipient phone number
  message: string;                    // SMS message text
  status: 'success' | 'failed';      // Delivery status

  // Twilio information
  twilioSid?: string;                 // Twilio's unique message ID (if sent)
  failureReason?: string;             // Error from Twilio (if failed)

  // Timestamps
  sentAt: Timestamp;                  // When Cloud Function processed it
}
```

**Example Document - Success**:
```
sentMessages/
  └── audit_doc_id_001
      ├── reminderId: "reminder_doc_id_456"
      ├── volunteerId: "user1"
      ├── eventId: "event_doc_id_789"
      ├── phoneNumber: "+14155552671"
      ├── message: "Reminder: Community Cleanup tomorrow at 9am!"
      ├── status: "success"
      ├── twilioSid: "SM1234567890abcdef1234567890abcdef"
      └── sentAt: Timestamp(1709996500, 0)
```

**Example Document - Failure**:
```
sentMessages/
  └── audit_doc_id_002
      ├── reminderId: "reminder_doc_id_457"
      ├── volunteerId: "user2"
      ├── eventId: "event_doc_id_789"
      ├── phoneNumber: "+15551234567"  // Invalid format
      ├── message: "Reminder: Community Cleanup tomorrow at 9am!"
      ├── status: "failed"
      ├── failureReason: "Invalid To phone number +15551234567"
      └── sentAt: Timestamp(1709996500, 0)
```

## Firestore Queries Used in App

### From Frontend (React)

#### Get volunteer profile
```typescript
getDoc(doc(db, 'volunteers', userId))
```

#### Get assigned events (for volunteer)
```typescript
query(
  collection(db, 'serviceEvents'),
  where('assignedVolunteers', 'array-contains', userId)
)
```

#### Get all volunteers (admin only)
```typescript
collection(db, 'volunteers')  // Security rules prevent non-admin reads
```

#### Get all events (admin only)
```typescript
collection(db, 'serviceEvents')  // Security rules prevent non-admin reads
```

### From Cloud Functions

#### Get pending reminders (every minute)
```typescript
query(
  collection(db, 'reminders'),
  where('status', '==', 'pending'),
  where('reminderTime', '<=', now),
  limit(100)
)
```

#### Get event with volunteers
```typescript
getDoc(doc(db, 'serviceEvents', eventId))
// Then read .data().assignedVolunteers
```

#### Get volunteer details (for phone number)
```typescript
getDoc(doc(db, 'volunteers', volunteerId))
// Read .data().phoneNumber
```

## Error Handling

### Frontend Errors

```typescript
try {
  // Cloud Function call
  const result = await createRemindersFunction({...});
} catch (error: any) {
  if (error.code === 'unauthenticated') {
    // User not logged in - redirect to login
  } else if (error.code === 'permission-denied') {
    // User not admin
  } else if (error.code === 'invalid-argument') {
    // Missing required fields
  } else if (error.code === 'internal') {
    // Other server error
    console.log(error.message);
  }
}
```

### Cloud Function Errors

```typescript
throw new functions.https.HttpsError(
  'error_code',
  'Human readable message'
);

// Valid error codes:
// - 'ok' (status 200)
// - 'cancelled' (status 1)
// - 'unknown' (status 2)
// - 'invalid-argument' (status 3)
// - 'deadline-exceeded' (status 4)
// - 'not-found' (status 5)
// - 'already-exists' (status 6)
// - 'permission-denied' (status 7)
// - 'unauthenticated' (status 16)
// - 'internal' (status 13)
// - 'unavailable' (status 14)
// - 'data-loss' (status 15)
```

## Batch Operations

The app uses Firestore batch writes for atomic multi-document updates:

```typescript
const batch = db.batch();

// Add multiple updates
batch.update(ref1, data1);
batch.update(ref2, data2);
batch.set(ref3, data3);

// Commit all at once
await batch.commit();
```

This ensures:
- Either all updates succeed or none
- No partial states
- Improved performance for multiple writes

## Field Naming Conventions

- **UI Fields**: camelCase (name, phoneNumber, eventDateTime)
- **System Fields**: camelCase (createdAt, updatedAt, isAdmin)
- **Array Fields**: Plural noun (assignedVolunteers, reminders)
- **Boolean Fields**: Prefix "is" or action verb (isAdmin, enabled, approved)
- **Timestamps**: Use Firebase Timestamp type (not string or number)
- **IDs**: uid, eventId, volunteerId, reminderId (explicit), or document ID

## Performance Indexes

Firestore automatically creates these as needed:

1. **Composite Index**: serviceEvents
   - assignedVolunteers (array)
   - eventDateTime (descending)
   - For: Finding events assigned to a specific volunteer, sorted by date

2. **Composite Index**: reminders
   - status (ascending)
   - reminderTime (ascending)
   - For: Finding pending reminders ready to send

If you see "Create index" prompts, click them in Firebase Console.

## Cost Estimation

### Storage
- Volunteer document: ~500 bytes
- Event document: ~1 KB
- Reminder document: ~300 bytes
- Sent message document: ~200 bytes

For 100 volunteers, 4 events/month, 1 reminder per event:
- ~150 KB/month
- $0.18/GB/month = **negligible cost**

### Reads
- Dashboard load: ~5-10 reads
- Create reminder: ~3-5 reads
- SMS send: ~3 reads per reminder

For 100 volunteers with 1 reminder each:
- ~400-500 reads/month
- $0.06/100K reads = **$0.024/month**

### Writes
- Create volunteer: 1 write
- Update profile: 1 write
- Create event: 1 write
- Create reminder: 1 write per volunteer
- Update reminder status: 1 write

For 100 volunteers with 1 reminder each:
- ~120 writes/month
- $0.18/100K writes = **$0.022/month**

### Total Firestore Cost
**~$0.05/month** for small deployment
